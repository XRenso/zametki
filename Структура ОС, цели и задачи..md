*Операционная система (универсальная)* - комплекс программ (ПО), обеспечивающая
* предоставление пользователю и прикладным программам доступ к API аппаратного обеспечения;
* распределение/управление ресурсами;
* абстракция от конкретных деталей ресурсов;
* управление пользователями

## Задачи
* Исполнение пользовательских программ;
* Управление аппаратными ресурсами;
	* Процессорное время + средства синхронизации и коммуникации;
	* Стандартный доступ к периферийным устройствам (*I/O устройства*);
	* Управление оперативной памятью (*распределение между процессами, организация виртуальной памяти*);
	* Загрузка программ в оперативную память и их выполнение;
	* Управление доступом к данным на энергонезависимых носителях (*прим. жесткие диски, оптические диски и тд.*), организованной в той или иной файловой системе;
	* Сохранение информации об ошибках системы;
* Обеспечение пользовательского интерфейса;
* Абстракция над аппаратными ресурсами;
	* Системный программный интерфейс (*API - application programming interface*);
	* Позволяет прикладному ПО работать на разных аппаратных ресурсах;


## Структура ОС
* Ядро;
	* планировщик задач (*если есть многозадачность*);
	* драйверы устройств (*в некоторых ядрах*) непосредственно управляющие оборудованием (*сетевая подсистема, файловая система*);
* Системные библиотеки (*инструментарий для совершения системных вызовов*);
*  Оболочка с утилитами;

*Ядро* - центральная часть операционной системы, обеспечивающая приложениям координированный доступ к ресурсам компьютера, таким как процессорное время, память, внешнее аппаратное обеспечение, внешнее устройство ввода и вывода информации. Также обычно ядро предоставляет сервисы файловой системы и сетевых протоколов.

*Системная библиотека* - сборщик подпрограмм или объектов, используемых для разработки ПО. С точки зрения ОС и прикладного ПО, библиотеки разделяются на динамические и статические:
* Статическая библиотека - объектный файл в виде файла (*нередко может быть поставлен вместе с исходным кодом*), код из которого выборочно или полностью вставляется в программу на этапе компоновки.
* Динамическая библиотека - файл, содержащий машинный код. Загружается в память процессора загрузчиком программ операционной системы, либо при создании процесса, либо по запросу работающего процесса, то есть динамически.

*Оболочка с утилитами* - совокупность специальных программ, предназначенных для облегчения общения пользователя с командами операционной системы.
