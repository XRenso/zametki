
#### 1. Jenkins
- **Что это?** Open-source инструмент для автоматизации CI/CD (Continuous Integration/Continuous Delivery). Сервер автоматизации, где строятся пайплайны для сборки, тестирования и деплоя кода.
- **Зачем?** Ускоряет разработку, минимизирует ошибки, интегрируется с Git, Docker и т.д.
- **Ключевые понятия:**
  - Pipeline: Скрипт (Jenkinsfile) на Groovy или declarative для этапов (build, test, deploy).
  - Job: Задача, которая запускается вручную или по триггеру (например, push в Git).
  - Plugins: Расширения для интеграций (e.g., Docker, Kubernetes).
- **Базовые шаги:**
  1. Установить Jenkins (via Docker или war-файл).
  2. Создать pipeline: `pipeline { agent any; stages { stage('Build') { steps { sh 'make' } } } }`.
- **Вопросы, которые могут задать:** Что такое CI/CD в Jenkins? Как настроить простой pipeline?

#### 2. Docker
- **Что это?** Платформа для контейнеризации приложений. Создаёт лёгкие, портативные контейнеры с кодом, runtime и зависимостями.
- **Зачем?** Обеспечивает consistency (работает везде одинаково), изоляцию, масштабируемость. "Build once, run anywhere".
- **Ключевые понятия:**
  - Image: Шаблон (readonly) для контейнера (e.g., Dockerfile: FROM ubuntu, RUN apt install...).
  - Container: Запущенный экземпляр image.
  - Docker Hub: Репозиторий для образов.
  - Volumes: Для persistent data.
  - Networks: Для связи контейнеров.
- **Базовые команды:**
  - `docker build -t myapp .` — собрать image.
  - `docker run -d -p 80:80 myapp` — запустить контейнер.
  - `docker ps` — список контейнеров.
  - `docker-compose up` — для multi-container apps (docker-compose.yml).
- **Вопросы:** Что такое контейнеризация? Разница между VM и контейнером (контейнеры делят kernel OS, VM — нет).

#### 3. Kubernetes (K8s)
- **Что это?** Open-source оркестратор контейнеров (от Google). Управляет кластером контейнеров: scaling, deployment, networking.
- **Зачем?** Автоматизирует деплой, scaling и управление apps в production. Подходит для microservices.
- **Ключевые понятия:**
  - Pod: Минимальная единица (1+ контейнеров).
  - Deployment: Управляет replicas pods.
  - Service: Экспозит pods (e.g., LoadBalancer).
  - Namespace: Логическая изоляция.
  - kubectl: CLI для управления (e.g., `kubectl apply -f deployment.yaml`).
- **Базовые шаги:**
  1. Установить minikube для локального тестирования.
  2. YAML-манифест: kind: Deployment, spec: replicas: 3.
  3. `kubectl get pods` — проверить статус.
- **Вопросы:** Что такое оркестрация? Разница Docker и K8s (Docker — для одного хоста, K8s — для кластера).

#### 4. Настройка CI/CD
- **Что это?** Continuous Integration (CI): Автоматическая сборка/тестирование при изменениях. Continuous Delivery/Deployment (CD): Автоматический деплой в staging/prod.
- **Зачем?** Ускоряет релиз, снижает риски, улучшает collaboration.
- **Ключевые понятия:**
  - Pipeline: Stages (build → test → deploy).
  - Triggers: Webhooks от Git (push, merge).
  - Artifacts: Результаты build (e.g., JAR, Docker image).
  - Tools: Jenkins/GitLab CI для реализации.
- **Базовые шаги в Jenkins/GitLab:**
  1. Настроить repo в Git.
  2. Создать .gitlab-ci.yml или Jenkinsfile.
  3. Stages: script: 'npm test', deploy to K8s via kubectl.
- **Вопросы:** Что входит в CI/CD пайплайн? Пример простого workflow.

#### 5. Сети (Networks)
- **Что это?** В DevOps — настройка networking для контейнеров, кластеров, CI/CD.
- **Зачем?** Обеспечивает связь, security, load balancing.
- **Ключевые понятия:**
  - IP, Ports, DNS: Базовые (e.g., TCP/UDP).
  - В Docker: Bridge (default), Host, Overlay (для swarm).
  - В K8s: Services (ClusterIP, NodePort), Ingress для external access.
  - Firewalls: iptables, security groups.
  - Load Balancing: Распределение трафика (e.g., NGINX, K8s LB).
- **Базовые команды:**
  - `docker network create mynet` — создать сеть.
  - `kubectl expose deployment myapp --type=LoadBalancer` — expose service.
  - `ip addr show` — проверить interfaces.
- **Вопросы:** Как контейнеры общаются? Что такое Ingress в K8s?

#### 6. GitLab
- **Что это?** Платформа для Git-репозиториев, CI/CD, issue tracking. Альтернатива GitHub.
- **Зачем?** Полный DevOps lifecycle: code → build → deploy.
- **Ключевые понятия:**
  - Repository: Хранение кода.
  - CI/CD: .gitlab-ci.yml для pipelines (stages, jobs).
  - Runners: Agents для выполнения jobs (shared или self-hosted).
  - Merge Requests: Code review.
- **Базовые шаги:**
  1. Создать проект.
  2. .gitlab-ci.yml: image: node, script: - npm install.
  3. `git push` — триггер pipeline.
- **Вопросы:** Разница GitLab и GitHub? Как настроить CI в GitLab?

#### 7. Ansible
- **Что это?** Tool для автоматизации конфигурации, deployment и orchestration. Agentless (via SSH).
- **Зачем?** Управляет серверами idempotently (повторно — без изменений), прост в использовании.
- **Ключевые понятия:**
  - Playbook: YAML-файл с tasks (e.g., install package).
  - Inventory: Список хостов (hosts: all).
  - Modules: Готовые задачи (e.g., apt, copy).
  - Roles: Переиспользуемые наборы playbooks.
- **Базовые команды:**
  - `ansible -i inventory all -m ping` — проверить связь.
  - `ansible-playbook site.yml` — запустить playbook.
  - Пример playbook: tasks: - name: Install nginx, apt: name: nginx state: present.
- **Вопросы:** Что такое configuration management? Разница Ansible и Puppet (Ansible push-based, без агентов).
